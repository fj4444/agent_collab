# 对工作计划的意见（MVP 版）

以下是面向 **最小可用产品（MVP）** 的精简审阅意见：保留“必须澄清/必须决策”的点，其余内容可延后。

## MVP 必须修正（不增加复杂度）
1. **修正标记拼写**：`[CHANGES_REQUIRED]}` 有多余 `}`，应为 `[CHANGES_REQUIRED]`，否则一致性判定会失效。
2. **澄清 workdir 语义**：`workdir` 仅用于存放 `.agent-collab/` 等产物，不应作为 agent 的执行目录。agent 的 cwd 应保持在项目根目录，否则测试与代码修改会跑到错误目录。

## 低成本补充（仍符合 MVP）
3. **执行阶段加最小安全护栏**：明确“每步执行前需用户确认”或“仅在用户显式确认后执行命令”，可避免误执行破坏性操作。

## 可以延后（非 MVP 必需）
- 并发编辑锁与合并策略
- debug log 轮转与大小管理
- 复杂 UI 交互（裁决对话框、多种快捷键）
- 结构化 front‑matter（版本/作者/时间）
- 完整 CLI 最小兼容矩阵（可先只支持一类 CLI）
- 复杂测试策略（MVP 可直接让 agent 选择测试命令）

## 对文档的最小补充建议
- 明确 **一致性判定** 规则（只保留一种）
- 明确 **文件路径约定**
- 描述 **session 恢复失败的最小降级**
- 描述 **危险命令的最小确认机制**
